#### new

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


// 以 new 操作符调用构造函数时


1) 创建一个空对象并且this变量引用了该对象, 同时还继承了该函数的原型

2) 属性和方法被加入到 this 引用的对象中

3) 新创建的对象由 this 所引用, 并且最后隐式返回 this( 若没有显示返回其他对象 )


//-------------------------------------------------------------------------------------------------------------------//


// 构造函数的返回值

当用 new 操作符创建对象时, 构造函数总是返回一个对象
( 默认返回是 this 所引用的对象 )


var Objectmaker = funcution () {

    // 构造函数可以返回任何值( 只要是对象 )
    ( 若返回非对象的值, 则该构造函数会忽略该值, 并返回默认 this 所引用的对象 )

    // 因为构造函数返回了that 则该 this.name 的属性会被忽略
    this.name = 'prop';
    
    // 创建并返回一个新对象, 则会忽略默认 this 返回的对象
    var that = {}
    that.name = '';
    return that;
};



// 若忘记使用 new 调用构造函数

可能导致逻辑和意外的行为发生

    - 导致构造函数中的 this 指向 window
    
    - 导致在构造函数 this.member 中创建的成员会默认添加到 window


//-------------------------------------------------------------------------------------------------------------------//


// 解决忘记使用 new 调用构造函数的问题


function Waffle() {

    // 判断 this 是否指向构造函数, 若未指向, 则使用 new 再次使用构造函数并返回
    if(!(this instanceof Waffle)) {
        return new Waffle();
    }

    // ES5严格模式中并不支持 argument.callee
    if(!this instanceof argument.callee ) {
        return new argument.callee();
    }
    
    this.name = 'alpha';
}


///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
```

## params

#### introduction

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


函数定义时不为参数指定的数据类型

函数上传递的参数不进行类型检查

函数不检查收到的参数个数


//-------------------------------------------------------------------------------------------------------------------//


// 参数是按值传递


参数，函数调用中，在函数的参数

JavaScript的参数通过传递价值：该功能只得到知道值，而不是争论的位置

如果一个函数改变一个参数的值，它不会改变参数的初始值

更改参数是不可见( 反射 )之外的功能 


//-------------------------------------------------------------------------------------------------------------------//


// 对象是通过引用传递


在 JavaScript 中, 对象引用值

正因为如此, 物体会像它们被传递参考：

如果一个函数改变一个对象的属性, 它改变了原有的价值

变更对象的属性是可见的( 反射 )的功能之外
```
